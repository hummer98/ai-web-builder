# AI Web Builder

[English version](README.md)

AI がコードを生成するだけでなく、開発者として振る舞う WYSIWYG Web サイトビルダー。稼働中のサイトを編集し、結果を視覚的に検証し、壊れたら自分で直す。

## なぜこれを作ったのか

AI Web サイトビルダーは世の中に溢れている — v0、bolt.new、Lovable、その他多数。それでもなぜ新しく作るのか？

既存ツールへの不満は3つに集約される。

**1. 「ここ」を指させない**

チャットだけのツール（v0、bolt.new）では、要素を言葉で説明するしかない：「ヒーローセクションの2番目のボタン」。GUI だけのツール（Wix、Squarespace）はドラッグ&ドロップできるが、すぐ限界がくる。AI Web Builder は両方を組み合わせる — ライブプレビュー上で要素をクリックし、AI に「これをこうして」と言える。AI はどのコンポーネントのどの行かを正確に把握している。

**2. AI はコードを生成するだけで検証しない**

ほとんどの AI コード生成ツールは出力したら終わり。AI Web Builder は編集のたびにフィードバックループを回す：Playwright でスクリーンショットを撮り、ブラウザコンソールを読み、サーバーログを確認し、壊れていたら自動で修正する — 最大3回リトライしてからユーザーに相談。ユーザーには常に動く画面だけが見える。

**3. サイトがプラットフォームに閉じ込められる**

Wix のサイトは Wix にしかない。Squarespace も同様。コード生成系のツールでさえ、独自のホスティングやプロジェクト形式に依存しがち。AI Web Builder は標準的な React + Hono のソースコードを出力し、あなたが所有する GitHub リポジトリにコミットする。`git clone` すれば、好きなツールで開発できる普通のプロジェクトが手に入る。

## 仕組み

```
ブラウザ                               Fly.io コンテナ
┌──────────────┐                     ┌──────────────────────────────────┐
│ チャット      │── WebSocket ──────→ │ Agent Server (:8080)             │
│ + プレビュー  │                     │   ↓                              │
│   (iframe)   │                     │ OpenCode AI エンジン (:4096)      │
│              │                     │   ├── ファイル編集                │
│  要素を       │                     │   ├── Playwright: スクリーンショット│
│  クリック →   │                     │   │   コンソール確認、視覚検証     │
│  AI が正確な  │                     │   └── ログリーダー: プロセス横断   │
│  コンポーネント│                     │       エラー検知                  │
│  を把握       │                     │   ↓                              │
│              │◄── HMR ────────────│ Vite (:5173) + Hono (:3000)     │
└──────────────┘                     └──────────────────────────────────┘
                                          │
                     ┌────────────────────┼────────────────────┐
                     ↓                    ↓                    ↓
               GitHub               Cloudflare            OpenRouter
             (バックアップ)         (本番ホスティング)       (LLM API)
```

1. **エディターを開く** — Fly Machine が自動起動
2. **プレビュー上で要素をクリック** — Source Locator がコンポーネントと行を特定
3. **メッセージを入力** —「これを青くして」「問い合わせフォームを追加して」「スマホ対応にして」
4. **AI がコードを編集** — HMR で即座に反映
5. **AI が自分の仕事を検証** — スクリーンショット → コンソール確認 → ログ確認 → 壊れていたら自動修正
6. **「公開」を押す** — Cloudflare Pages + Workers にデプロイ
7. **タブを閉じる** — マシン停止、課金停止

## 主要な設計判断

| 判断 | 理由 |
|------|------|
| **標準的な React + Hono を出力** | 独自形式なし。`git clone` すれば普通のプロジェクト。 |
| **ユーザーごとに1コンテナ** | 各ユーザーが完全な開発環境（Vite、Hono、OpenCode）を持つ。コストはかかるが、本物の HMR と AI デバッグが可能になる。スケールではなく品質のための設計。 |
| **AI 自己修復ループ** | 静的解析 → Playwright 視覚検証 → コンソール/ログ確認 → 自動リトライ。ユーザーに壊れた画面を見せない。 |
| **クリック選択 + チャット** | GUI でもチャットでもない。クリックが AI に空間的コンテキストを、チャットが表現力を与える。 |
| **テンプレートなし** | AI が毎回ゼロから生成。テンプレートは制約になる。優秀な AI には不要。 |
| **GitHub を信頼の源泉に** | AI の編集は全て自動コミット。完全な履歴、簡単な差し戻し、データ消失なし。 |

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | React + Tailwind CSS |
| バックエンド | Hono（開発: Node.js / 本番: Cloudflare Workers） |
| データベース | Cloudflare D1（SQLite 互換） |
| AI エンジン | [OpenCode](https://opencode.ai) サーバーモード |
| LLM | OpenRouter 経由（Gemini、GPT、Kimi 等） |
| 編集環境 | Fly.io（Machines + Volumes、自動起動/停止） |
| 本番環境 | Cloudflare Pages + Workers |
| Git | GitHub App（`ai-web-builder[bot]`）による自動コミット |

## リポジトリ構成

```
ai-web-builder/
├── editor/                 エディター UI（チャット + iframe シェル）
├── container/
│   ├── agent-server/       WebSocket ↔ OpenCode 橋渡し
│   ├── log-reader-mcp/     プロセス横断ログリーダー（MCP サーバー）
│   └── scaffold/           ゲストサイトの初期ファイル一式
├── e2e/                    Playwright E2E + デモスクリプト
├── docs/                   設計ドキュメント
├── Dockerfile
└── fly.toml
```

ビルダーはユーザーごとに**別のゲストリポジトリ**を編集する — GitHub 上の標準的な React + Hono プロジェクトで、Cloudflare にデプロイされる。

## ローカル開発

```bash
direnv allow    # .envrc のシークレット読み込み
npm run dev     # 4プロセス並列起動:
                #   Agent Server     :8080
                #   opencode serve   :4096
                #   Vite dev server  :5173
                #   Hono dev server  :3000
```

## これは何ではないか

- **SaaS ではない。** 特定のユーザー（サイトが欲しい友人たち）のために作った個人ツール。スケールは想定していない。
- **テンプレートエンジンではない。** AI が毎回ゼロから生成する。
- **ホスト型 IDE ではない。** AI が開発者で、ユーザーがクリエイティブディレクターとなる WYSIWYG エディター。
