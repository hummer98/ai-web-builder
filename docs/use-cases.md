# ユースケース一覧（非エンジニア向け Web サイト構築）

非エンジニアの友人が「自分のサイトを作りたい・直したい」と思ったときに発生するユースケースを網羅的に整理し、現在の実装状況を評価する。

---

## 凡例

| 記号 | 意味 |
|------|------|
| ✅ | E2E で動作する |
| ⚠️ | 部分的に実装済み（配線不足・UI 不足等） |
| ❌ | 未実装 |

---

## 1. サイトを始める

| # | ユースケース | 具体的な操作イメージ | 状態 | 備考 |
|---|------------|-------------------|------|------|
| 1.1 | エディターを開く | URL にアクセスするだけ | ✅ | Fly Auto Start で自動起動、Cloudflare Access 認証済み |
| 1.2 | 新しいサイトを作る | 「カフェのサイトを作って」 | ⚠️ | WS ハンドラー (`create-site`) は実装済みだが、チャットからの自然言語トリガーなし。UI ボタンもなし |
| 1.3 | 既存サイトを編集する | 「le-serpent を編集したい」 | ⚠️ | `import-repo` ハンドラー実装済み。コンテナ起動時に `sites.json` + `SITE_DOMAIN` で自動 clone は動作する。チャットからの動的切替は不可 |
| 1.4 | 複数サイトを切り替える | 「別のサイトに切り替えたい」 | ❌ | 1コンテナ1サイト方式。切替 UI なし |

## 優先度の高い未実装ユースケース

実際の利用で最初にブロッカーになりそうなものを優先度順に整理:

### P0（これがないと基本フローが成立しない）

| # | ユースケース | 理由 |
|---|------------|------|
| 8.1 | Undo ボタン | WS ハンドラーは実装済みなのに UI がない。非エンジニアは「元に戻す」が最も重要な安心装置 |
| 1.2 | チャットからサイト作成 | 初回体験のエントリーポイント。現在は WS メッセージ手動送信が必要 |

### P1（利用中に高頻度で発生する）

| # | ユースケース | 理由 |
|---|------------|------|
| 4.1 | 画像アップロード | 画像を扱えないとサイト構築の大半が詰まる |
| 5.1 | マルチページ | 1ページだけのサイトは用途が限られる |
| 9.7 | ストリーミング応答 | 待ち時間の体感が大きく UX に影響 |

### P2（あると嬉しい）

| # | ユースケース | 理由 |
|---|------------|------|
| 6.1 | フォームのバックエンド | フロントだけでは問い合わせが受けられない |
| 8.3 | 変更履歴表示 | 安心感と振り返り |
| 3.6 | レスポンシブ指示 | スマホ表示の品質に直結 |
